# 口播视频稿生成技能

## 简介

基于 JustRL 论文（arXiv ID: 2512.16649）的核心数据，提供 **10 分钟口播视频稿生成**的完整解决方案。

**核心能力**：
- 深度阅读技术文档，提取核心信息
- 转化为易于理解的内容
- 生成完整的视频时间线和画面描述

---

## 工作流程

### 阶段一：文档获取与分析 (Document Analysis)

1. **获取论文全文**
   - 工具：`web_fetch`
   - 输入：arXiv 链接（如 https://arxiv.org/abs/2512.16649）
   - 目标：获取 PDF 或 HTML 全文

2. **深度阅读论文**
   - 读取 Abstract
   - 提取核心公式（如 `Accuracy = 28% -> 54.9%`）
   - 理解实验设置（如 `DeepSeek-R1-Distill-Qwen-1.5B`）
   - 记录关键结论（如 `SOTA performance with 50% compute`）

### 阶段二：脚本架构设计 (Script Architecture)

1. **确定叙事风格**
   - 风格选项：
     - A. 技术硬核（专业、数据驱动）
     - B. 温暖叙事（文科生视角、人性化）
     - C. 爆款包装（有煽动性）

2. **构建时间轴**（10分钟标准）
   - 开场：30秒 - 主题引入
   - 核心解读：2-4分钟 - 技术原理解析
   - 数据亮点：1-2分钟 - 震撼数据展示
   - 理论批判：2分钟 - 对复杂性的反思
   - 实践应用：1.5分钟 - 具体案例与方法
   - 结尾：1分钟 - 理念升华与号召

### 阶段三：内容撰写 (Content Creation)

1. **撰写旁白文案**
   - 标准语速：180-200 字/分钟
   - 语调选择：温暖、调侃、专业、总结性
   - 内容要求：
     - 技术准确性（基于论文数据）
     - 通俗性（避免过多术语）
     - 趣味感（加入个人观点、故事化）
     - 节奏感（短句、幽默、反问）

2. **设计画面描述**
   - 风格：极简主义（纯白底、黑字、手绘元素）
   - 元素：便签纸、思维导图、数据图表、黑板、嫩芽
   - 动画：淡入淡出、脉动、弹跳、闪烁

### 阶段四：制作指南生成 (Production Guide)

1. **视觉设计规范**
   - 字体：思源黑体 (Source Han Sans CN)
   - 大小：36pt (标题)、24pt (正文)
   - 色彩：黑 (#000000)、米色 (#F5F5DC)、暖灰 (#F0F0F0)

2. **音频制作规范**
   - 语速：180-200 字/分钟
   - 语调：温暖、平静、有力
   - 背景音乐：极简电子音乐 (Lo-fi)

3. **剪辑与导出**
   - 分辨率：1920x1080 (16:9)
   - 编码：H.264
   - 帧率：30fps
   - 软件：Adobe Premiere Pro 或 DaVinci Resolve

## 设计规范

### 视觉元素
- **背景：** 纯白 (#FFFFFF)
- **主色：** 黑 (#000000)
- **辅助色：** 米色 (#F5F5DC)、暖灰 (#F0F0F0)、淡金 (#FFD700)
- **元素：** 便签纸、思维导图、数据图表、嫩芽

### 动画风格

- **转场：** Ease-In-Out 缓动 (0.5-1秒)
- **特效：** 弹跳、脉动、闪烁
- **节奏：** 动作与旁白同步

### 字体规范
- **标题：** 思源黑体 - Bold (粗体)
- **正文：** 思源黑体 - Regular (常规)
- **代码/公式：** Courier New (等宽字体)

### 音频规范
- **语速：** 180-200 字/分钟
- **语调：** 温暖、平静、略带调侃
- **背景音乐：** 极简电子音乐 (Lo-fi)、节奏舒缓

## 使用说明

1. **基于论文创作**
   - 确保所有技术数据（如准确率、节省算力）准确无误
   - 避免过度解读，保持客观性

2. **人味化包装**
   - 将技术硬核内容用温暖、有温度的方式传达
   - 使用比喻和类比 (如 "像做蛋糕"、"像跑步心率")
   - 加入文科生视角 (如 "没有技术背景也能抓住机遇")

3. **节奏控制**
   - 严格遵循 10 分钟时间轴
   - 每个段落时长精确到秒
   - 旁白语速稳定在 180-200 字/分钟

4. **视觉一致性**
   - 使用统一的字体、颜色和动画风格
   - 避免生硬转场，使用淡入淡出

## 常见问题

**Q: 如何处理没有公式的论文？**
A: 使用 web_fetch 配合正则表达式提取关键信息，或使用专门的论文解析 API。

**Q: 能处理多语言内容吗?**
A: 目前支持中文和英文，可通过修改 prompt 实现其他语言。

**Q: 如何确保技术准确性?**
A: 在 prompt 中强调"严格基于原始论文内容，不要添加论文中没有的数据"，使用准确的术语和公式。

**Q: 支持实时生成吗?**
A: 当前版本为批处理模式，建议使用异步 API 或 Web Socket 实现实时性。
